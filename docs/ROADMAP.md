## Phase 2: 데이터 강화

### 2.1 실제 토큰 사용량 추적

**현재 한계:** 프롬프트 길이 기반 추정치만 제공

**개선 방안:**
- Claude API 응답의 `usage` 필드에서 실제 토큰 수 추출
- 세션별, 프로젝트별 토큰 소비량 정확히 추적
- 비용 추정 기능 추가

```python
# 예시 데이터 구조
{
  "input_tokens": 1500,
  "output_tokens": 4500,
  "total_tokens": 6000,
  "estimated_cost": 0.045  # USD
}
```

### 2.2 응답 품질 분석

**목표:** Claude의 응답이 얼마나 도움이 되었는지 분석

**구현 아이디어:**
- 응답 후 사용자 행동 패턴 분석
  - 즉시 다음 작업 진행 → 만족
  - 같은 질문 반복 → 불만족
  - 수정 요청 → 부분 만족
- "좋아", "감사", "고마워" 등 긍정 피드백 감지
- 오류 재시도 패턴 감지

### 2.3 컨텍스트 효율성 분석

**목표:** 컨텍스트 윈도우 사용 효율 분석

**분석 항목:**
- 세션당 평균 컨텍스트 사용량
- `/compact` 사용 빈도 및 효과
- 컨텍스트 오버플로우 발생 빈도

## Phase 3: 고급 분석

### 3.1 AI 페어 프로그래밍 효율성 점수

**목표:** Claude와의 협업 효율성을 수치화

**측정 지표:**
- 작업 완료율: 시작한 작업 대비 완료 비율
- 반복 요청률: 같은 내용 재요청 비율
- 수정 요청률: 생성된 코드 수정 요청 비율
- 세션 생산성: 세션당 완료된 작업 수

```
효율성 점수 = (완료율 × 0.4) + ((1 - 반복률) × 0.3) + ((1 - 수정률) × 0.3)
```

### 3.2 프로젝트 복잡도 분석

**목표:** 프로젝트별 작업 복잡도 파악

**분석 요소:**
- 세션당 평균 프롬프트 수
- 평균 프롬프트 길이
- 오류 언급 빈도
- 디버깅 관련 프롬프트 비율

### 3.3 학습 곡선 분석

**목표:** 시간에 따른 Claude 활용 능력 성장 추적

**측정 항목:**
- 프롬프트 품질 점수 변화
- 작업 완료 속도 변화
- 반복/수정 요청 감소율
- 고급 기능 활용 증가율

## Phase 4: 자동화 강화

### 4.1 자동 Skill 생성

**현재:** 추천만 제공, 사용자가 직접 생성

**개선:**
- 반복 패턴 감지 시 자동으로 skill 파일 생성
- 사용자 승인 후 적용

```bash
/cc-insights:skills auto-generate
# → 3개의 skill 자동 생성됨:
#   - /commit (150회 반복)
#   - /handoff (65회 반복)
#   - /log-check (22회 반복)
# 적용하시겠습니까? [Y/n]
```

### 4.2 주간/월간 자동 리포트

**목표:** 정기적으로 리포트 자동 생성 및 저장

**구현:**
- cron 또는 launchd를 통한 스케줄링
- 마크다운 리포트 자동 저장
- 이메일/Slack 알림 (선택)

```bash
/cc-insights:schedule weekly
# → 매주 일요일 자정에 주간 리포트 생성
```

### 4.3 실시간 경고 시스템

**목표:** 비효율적인 패턴 즉시 감지 및 알림

**경고 트리거:**
- 같은 프롬프트 3회 이상 반복
- `/clear` 과다 사용 (5회/일 초과)
- 품질 점수 급격한 하락
- 비정상적으로 긴 세션

## Phase 5: 시각화 및 대시보드

### 5.1 터미널 대시보드

**목표:** 터미널에서 실시간 대시보드 제공

**기술 스택:**
- Rich (Python) 또는 blessed (Node.js)
- 실시간 업데이트

```
┌─────────────────────────────────────────────────────────────┐
│  cc-insights Dashboard                         [09:30:45]   │
├─────────────────────────────────────────────────────────────┤
│  Today: 45 prompts | Quality: 8.5 | Projects: 3            │
├─────────────────────────────────────────────────────────────┤
│  Hourly Activity                                            │
│  ▁▂▃▄▅▆▇█▇▆▅▄▃▂▁                                           │
│  0  3  6  9  12 15 18 21 24                                │
├─────────────────────────────────────────────────────────────┤
│  Active Project: alphaswarm                                 │
│  Session: 2h 15m | Prompts: 28                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 웹 대시보드

**목표:** 브라우저에서 시각적 대시보드 제공

**기술 스택:**
- Frontend: React + Recharts
- Backend: FastAPI (Python)
- 로컬 서버로 실행

**기능:**
- 인터랙티브 차트
- 기간 필터링
- 프로젝트별 드릴다운
- 트렌드 시각화

### 5.3 내보내기 기능

**목표:** 다양한 형식으로 데이터 내보내기

**지원 형식:**
- CSV: 스프레드시트 분석용
- JSON: 프로그래밍 활용
- PDF: 공유용 리포트
- PNG: 차트 이미지
